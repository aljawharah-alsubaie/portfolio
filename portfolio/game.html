Find Differences Game

A multiplayer networked game where players compete in real-time to spot differences between two similar images. The game features server-client architecture with socket programming, waiting room management, live scoring system, and multiple game rounds with increasing difficulty.

Java Socket Programming Networking Multi-threading Java Swing Client-Server

üìñ Game Overview

Find Differences is an interactive multiplayer game built with Java networking concepts. Players connect to a central server, enter a waiting room, and compete to identify differences between two nearly identical images. The game demonstrates real-time communication, concurrent user handling, and synchronized gameplay across multiple clients.

The server manages up to 4 players simultaneously, displaying two kitchen scene images with subtle differences such as missing objects, color changes, repositioned items, and altered details. Players carefully examine both images and submit their answer - the total number of differences they discovered. Correct answers earn points, and the player with the highest score wins!

üíª Technical Implementation

The game leverages Java's networking capabilities to create a scalable multiplayer experience with clean separation of concerns.

Server Architecture

The server acts as the central coordinator, managing client connections, maintaining player lists, synchronizing game state, and broadcasting updates. It uses ServerSocket to listen on port 9090 and spawns a new NewClient thread for each connection. The server tracks two key lists: connected clients (all users) and active players (those in waiting room/game).

Client Architecture

Clients connect using Socket to establish communication channels (BufferedReader for input, PrintWriter for output). The ServerConnection class runs in a separate thread to continuously listen for server messages without blocking the UI. Multiple GUI frames handle different game states: FirstFrame (connection), WaitingRoom (lobby), game (gameplay), and Last (results).

Communication Protocol

Server and clients exchange messages using a custom text-based protocol. Key messages include player names, "tryToJoinRoom" requests, "isNotFull"/"isFull" responses, player list updates, and game state changes. All communication is text-based over TCP sockets, ensuring reliable, ordered delivery of game data.
