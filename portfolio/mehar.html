Architecture

Client-Server Architecture: The platform follows a traditional client-server model with clear separation of concerns. The frontend (HTML/CSS/JavaScript) handles user interactions and presentation, while the backend (PHP) manages business logic, authentication, and database operations. This architecture provides scalability, maintainability, and enhanced security for user data.

Database Design

The MySQL database implements a normalized relational schema with 9 core entities including Users, Workshops, Bookings, Reviews, Posts, Likes, Favorites, Workshop_Schedule, and Previous_Works. The design ensures data integrity through proper foreign key relationships and supports complex queries for features like personalized recommendations and community interactions.
